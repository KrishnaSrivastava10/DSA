SET (Java)
- Set is a collection in Java that does not allow duplicate elements
- It is mainly used to store unique values

Why use Set?
- To remove duplicates from array or list
- To store unique data only

Set<Integer> set = new HashSet<>();
- Example
int[] arr = {1, 1, 2, 2, 3};
Set<Integer> set = new HashSet<>();
for (int x : arr) {
    set.add(x);
}

Important Note
- Using Set is easy but not optimal for sorted arrays


HashSet vs LinkedHashSet (Removing Duplicates)
❌ HashSet

Does not maintain order
Stores elements based on hashing
Output order may be random
Removes duplicates ✅

Example
Set<Integer> set = new HashSet<>();
int[] arr = {10, 23, 23, 27};
for (int x : arr) {
    set.add(x);
}
System.out.println(set);
// Output order not guaranteed

✅ LinkedHashSet
Maintains insertion order
Removes duplicates automatically
Best choice when order must be preserved

Example
Set<Integer> set = new LinkedHashSet<>();
int[] arr = {10, 23, 23, 27};
for (int x : arr) {
    set.add(x);
}
System.out.println(set);
// Output: [10, 23, 27]

Key Point (Interview)
Use HashSet → when order does not matter
Use LinkedHashSet → when order must be preserved
For sorted array problems → prefer LinkedHashSet

ARRAY LIST (Java)
- ArrayList is a dynamic array in Java
- It is part of Collection Framework
- Located in java.util package
- Size can grow or shrink automatically

ArrayList<Integer> list = new ArrayList<>();
- Example
ArrayList<Integer> list = new ArrayList<>();
list.add(10);
list.add(20);
list.add(30);
System.out.println(list.get(1)); // 20

What ArrayList does
- Stores elements in index-based order
- Allows duplicate values
- Allows random access using index
- Internally uses an array

| Feature           | Array                         | ArrayList                   |
| ----------------- | ----------------------------- | --------------------------- |
| Size              | Fixed size                    | Dynamic size                |
| Part of           | Language feature              | Collection Framework        |
| Can grow          | ❌ No                          | ✅ Yes                       |
| Stores primitives | ✅ Yes                         | ❌ No (uses wrapper classes) |
| Stores objects    | ✅ Yes                         | ✅ Yes                       |
| Built-in methods  | ❌ Very few                    | ✅ Many (add, remove, size)  |
| Type safety       | Can store mixed (if Object[]) | Generic (type-safe)         |
